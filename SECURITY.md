# Security Policy

## Supported Versions

The following versions of this project currently receive security updates and patches:

| Version                 | Supported |
| ----------------------- | --------- |
| Latest (main branch)    | ✅         |
| Previous stable release | ✅         |
| Older releases          | ❌         |

We strongly recommend that all users upgrade to the latest supported version to receive security fixes and improvements.

---

## Reporting a Vulnerability

We take security vulnerabilities seriously and appreciate responsible disclosure.

### Reporting Process

If you discover a security vulnerability, please **do not create a public GitHub issue**.

Instead, report it through one of the following methods:

* GitHub Security Advisories (preferred)
* Email: [electrokatt.co@gmail.](mailto:security@yourdomain.com)
* [com](mailto:security@yourdomain.com)
* Project Maintainer Contact: +27 62 731 3073

### What to Include

Please provide:

* A description of the vulnerability
* Steps to reproduce the issue
* Potential impact
* Suggested remediation (if available)
* Screenshots, logs, or proof-of-concept code where applicable

### Response Timeline

| Stage                   | Expected Time         |
| ----------------------- | --------------------- |
| Initial acknowledgement | Within 48 hours       |
| Status update           | Within 7 days         |
| Resolution target       | Depending on severity |

### Responsible Disclosure

Please allow reasonable time for investigation and remediation before publicly disclosing the vulnerability.

### Accepted Vulnerabilities

If the vulnerability is confirmed:

* The issue will be prioritized based on severity.
* A security patch will be developed and tested.
* A security advisory may be published after the fix is released.
* Reporters may be credited for their contribution unless anonymity is requested.

### Invalid Reports

The following generally do not qualify as security vulnerabilities:

* Issues requiring physical access to a user's device
* Social engineering attacks
* Outdated browser vulnerabilities
* Reports lacking sufficient reproduction details

Thank you for helping keep this project secure.
